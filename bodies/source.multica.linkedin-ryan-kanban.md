# Ryan Steckler 的 Multica Kanban 工作流

**证据等级：S3，公开用户自报。**

Ryan Steckler（Intuit Credit Karma CPO，曾参与 Mint/TurboTax）称自托管 Multica 后放弃自己写的 orchestrators，用 Kanban 让 designer、engineer、copywriter、reviewer、GitOps/deployment Agent 依次处理任务。

他认为可检查性、控制和并行工作是主要价值，同时明确 token 使用上升。单个 power-user 样本不能代表一般团队留存或 ROI。关联：[[company.multica]]。
