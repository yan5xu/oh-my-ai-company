# Viktor homepage

采集时间：2026-07-08。来源：browser read，正文可用。

官网首页把 Viktor 定义为 Slack 和 Microsoft Teams 里的 AI employee，连接 3,200+ tools，能产出 finished deck、dashboard、app，而不是建议文本。核心文案是 “A chatbot answers. Viktor delivers.”

官网还强调：

- one message, whole stack；
- daily reports / weekly digests / scheduled tasks；
- 记住公司决策和偏好；
- 敏感/不可逆动作前需要 approval；
- invited channel 才读内容；
- pricing 从 $100 credits 免费开始，之后 starts $50/month；
- 首页写 “Trusted by 40,000+ teams”。

关联公司：[[company.viktor]]。

## 2026-07-20 FAQ 补充核验

本轮直接读取首页 FAQ 与结构化页面内容，补充以下边界：

- 每个 Viktor 有持久 cloud computer / workspace。
- 它从被邀请进入的频道和对话中学习，并把流程沉淀为内部 skills / notes。
- 当前 FAQ 说 integrations 在团队内共享，访问控制主要是 workspace-level；per-user scoping、RBAC 和 private mode 仍在 roadmap。
- 断开 integration 会停止新数据访问，但派生的 notes、summaries 和 context 可能继续存在。
- Clean Workspace 和完整 workspace data wipe 可删除派生状态。

口径冲突：2026 年 2 月 changelog 同时声称 per-user channel privacy 和私人频道/DM 隔离。可能是频道可见性、集成权限和派生上下文属于不同层，也可能是页面版本不一致；账号内未验证前不合并为完整逐人隔离能力。
