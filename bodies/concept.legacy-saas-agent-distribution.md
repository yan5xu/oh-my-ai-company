# 旧 SaaS 作为 Agent 新业务的现金流、上下文与分发底座
已有 SaaS 在 Agent 时代不只是旧产品。它可能同时提供四类资产：

1. 可持续现金流与融资叙事；
2. 已经存在的客户和获客渠道；
3. 项目、任务、文档、日历等业务上下文；
4. 工程、数据、安全与实施团队。

[[company.motion]] 到 [[company.wonderly]] 是当前代表案例：Motion 的自动排程与 work suite 仍有数十万级月访问估算和成熟付费获客；Wonderly 复用同一法人、创始人与多名员工，进入 service SMB 的收入闭环。[[source.similarweb.motion-2026-h1]] [[source.linkedin.wonderly-company-2026-07-15]]

但旧 SaaS 也是约束：旧用户会追问功能是否维护、是否被当作新产品 beta、订阅收入是否被转投、迁移是否公平。研究时必须同时看资产复用与 trust debt，不能把 pivot 只写成增长故事。
