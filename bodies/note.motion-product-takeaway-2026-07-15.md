# Motion 产品判断：已有 SaaS 不是历史包袱，而是新 Agent 业务的底座与约束
## CP Takeaway

Motion 最值得学习的不是“从日历转 AI Employees”，而是它持续把已有资产重组为更大的产品。

早期 Chrome 扩展先解决注意力，后来自动排程解决“下一步做什么”，AI Employees 再尝试直接做工作。当前主站回到 SuperApp，并不一定意味着此前员工叙事完全失败，更可能是把有效能力重新收进更稳定、可计费的套件。

真正的优势是：Motion 已有项目、任务、日历、文档、客户和成熟买量渠道。做 Agent 时，这些既是上下文，也是分发。新产品不是从零开始。

但这也提醒我：旧 SaaS 的收入与用户信任不是免费的。用户会感受到功能弃用、产品注意力转移和被当作 beta。团队必须同时管理 innovation 与 trust debt。

[[company.wonderly]] 更进一步：它不再卖员工名字，而是卖 service SMB 的收入闭环。对我们而言，研究老产品时要问的不只是“会不会被 Agent 替代”，还要问“它能不能成为 Agent 新业务的现金流、上下文和分发底座”。
