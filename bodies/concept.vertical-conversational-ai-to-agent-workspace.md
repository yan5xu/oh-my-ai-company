# 从对话自动化上移到垂直 Agent 工作空间

一种 AI employee 产品演进路径：成熟的语音、客服或对话自动化厂商，利用既有行业数据、系统集成、合规流程与实施团队，把单点机器人扩展为岗位化 AI 同事、内部协作工作区和多 Agent 工作流。

[[company.floatbot]] 是当前样本：早期做银行/政府/BFSI 的 Chatbot 与 Voicebot，随后推出 Agent M、RAG Search 和实时辅助，最终用 FloatSpace 把 ADDI、LEXI、AURA 等角色放进保险工作空间。[[source.floatbot.about]] [[source.floatbot.floatspace]]

这个模式的判断重点不是是否用了“AI employee”命名，而是：

1. 是否已经连接系统 of record 并可读写真实业务数据；
2. 是否拥有可复用的行业规则、模板、评测和合规边界；
3. 是否能完成从客户交互到后台动作的闭环；
4. 实施成本能否逐步从项目制压缩为产品化部署。

Floatbot 案例中的 16 周上线、UAT、幻觉修复和客户共同投入，说明这条路线拥有较强交付壁垒，但当前仍明显依赖 FDE/服务。[[source.floatbot.case-mountain-peak]] [[source.floatbot.case-capital-recovery]]
