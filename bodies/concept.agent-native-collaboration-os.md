# Agent-native collaboration OS

一种把 AI Agent 作为持续组织成员，而不是一次性工具或聊天 bot 的产品模式。

## 最小构件

- 持续身份与长期上下文；
- 频道、线程、DM 等共享协作空间；
- task ownership、handoff、review 与 reminder；
- 可替换的模型/runtime；
- 权限、凭据、审批、trace 与人类 gate；
- Agent 可使用的 app identity 与 action contract。

[[company.raft]] 的差异在于从用户已有的本地 runtime 进入，以 Agent Experience 处理 turn-based agents 的 inbox、过时草稿和责任冲突。[[company.helio]] 则更偏从可被雇用的 AI teammate、业务职责和组织工作区进入。

## 核心问题

该模式是否优于 hierarchical orchestrator，不能由 Agent 数量或消息量回答。应比较 accepted deliverables、handoff success、human interventions、cost per accepted output 与 time-to-verification。

当前为 active / medium confidence：产品能力已存在，但一般团队的长期价值和商业采用仍需更多公司与独立数据验证。
