# 商户侧 Agent Readiness

## 定义

把现有产品改造成 Agent **可发现、可调用、可授权、可按量付费、可交付、可审计** 的供给侧能力，而不只是给 Agent 一张卡或一个 wallet。

## 为什么重要

Agent commerce 有双边冷启动：Agent 没有可购买供给时，钱包没有用途；商户看不到 Agent demand 时，也不会专门接协议。Merchant readiness layer 试图降低商户改造成本，并让同一份产品同时服务人类和 Agent。

[[company.clink]] 用 MoR、Billing、Skill review、Agent Payment 和 coding-agent integration 做这件事；[[company.skyfire]] 从 Seller Service、KYA 与 checkout 建接受面；[[company.sapiom]] 从 Agent 采购资源与控制平面连接供给。

## 待验证

- Skill / service discovery 是否会被通用协议或大平台吸收；
- 商户是否真的获得新增 Agent revenue，而不只是改造成本；
- verified marketplace 是否能形成供给密度与复购；
- Visa/Mastercard/Stripe 等网络层能否直接标准化并压缩独立平台空间。

当前状态为 candidate，需要更多公司与真实交易数据验证。
